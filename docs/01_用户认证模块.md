# 用户认证模块

## 模块简介

用户认证模块是整个校园服务平台的基础，负责用户的注册、登录、权限验证等功能，采用 JWT Token 进行身份认证。

## 功能清单

### 1. 用户注册
- 手机号注册
- 密码设置
- 昵称设置
- 默认角色：普通用户

### 2. 用户登录
- 手机号 + 密码登录
- 返回 JWT Token
- Token 包含用户 ID 和角色信息

### 3. 权限控制
- 普通用户（USER）
- 管理员（ADMIN）
- 基于角色的接口访问控制

### 登录 Controller
```java
/**
 * 用户登录
 * @param userDTO 包含手机号和密码
 * @return 登录成功返回 JWT Token
 */
@PostMapping("/login")
public Result<String> login(@RequestBody UserDTO userDTO) {
    // 1. 参数校验
    if (userDTO.getPhone() == null || userDTO.getPassword() == null) {
        throw new ParameterValidationException("手机号和密码不能为空");
    }

    // 2. 查询用户
    User user = userService.login(userDTO);

    // 3. 生成 Token
    Map<String, Object> claims = new HashMap<>();
    claims.put(JwtClaimsConstant.USER_ID, user.getId());
    claims.put(JwtClaimsConstant.USERNAME, user.getNickname());
    claims.put(JwtClaimsConstant.ROLE, user.getRole());

    String token = JwtUtil.createJWT(
        jwtProperties.getAdminSecretKey(),
        jwtProperties.getTtl(),
        claims
    );

    return Result.success(token);
}
```

### JWT 工具类
```java
/**
 * JWT 工具类
 * 用于生成和解析 JSON Web Token
 */
public class JwtUtil {
    // Token 过期时间：24小时
    public static final long JWT_TTL = 24 * 60 * 60 * 1000L;

    /**
     * 创建 JWT Token
     * @param secretKey 密钥
     * @param ttlMillis 过期时间（毫秒）
     * @param claims 自定义载荷
     * @return Token 字符串
     */
    public static String createJWT(String secretKey, long ttlMillis, Map<String, Object> claims) {
        // 1. 指定签名算法
        SignatureAlgorithm signatureAlgorithm = SignatureAlgorithm.HS256;

        // 2. 设置过期时间
        long expMillis = System.currentTimeMillis() + ttlMillis;
        Date exp = new Date(expMillis);

        // 3. 创建 JWT Builder
        JwtBuilder builder = Jwts.builder()
                .setClaims(claims)
                .signWith(signatureAlgorithm, secretKey)
                .setExpiration(exp);

        return builder.compact();
    }

    /**
     * 解析 JWT Token
     * @param secretKey 密钥
     * @param token Token 字符串
     * @return 载荷内容
     */
    public static Claims parseJWT(String secretKey, String token) {
        return Jwts.parser()
                .setSigningKey(secretKey)
                .parseClaimsJws(token)
                .getBody();
    }
}
```

### 登录拦截器
```java
/**
 * JWT Token 验证拦截器
 * 用于验证请求中的 Token 是否有效
 */
@Component
public class JwtTokenUserInterceptor implements HandlerInterceptor {

    @Autowired
    private JwtProperties jwtProperties;

    /**
     * 前置处理：在 Controller 执行前验证 Token
     */
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) {
        // 1. 判断是否为接口资源（放行静态资源）
        if (!(handler instanceof HandlerMethod)) {
            return true;
        }

        // 2. 从请求头获取 Token
        String token = request.getHeader(jwtProperties.getUserTokenName());

        // 3. 验证 Token
        try {
            Claims claims = JwtUtil.parseJWT(jwtProperties.getUserSecretKey(), token);

            // 4. 将用户信息存入线程上下文
            Long userId = Long.valueOf(claims.get(JwtClaimsConstant.USER_ID).toString());
            BaseContext.setCurrentId(userId);

            return true;
        } catch (Exception ex) {
            // Token 无效或过期
            throw new UserNotLoginException();
        }
    }

    /**
     * 后置处理：清理线程上下文
     */
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) {
        BaseContext.removeCurrentId();
    }
}
```

```sql
CREATE TABLE `user_info` (
  `id` int NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `phone` varchar(20) NOT NULL COMMENT '手机号',
  `password` varchar(200) NOT NULL COMMENT '密码',
  `nickname` varchar(50) DEFAULT NULL COMMENT '昵称',
  `avatar` varchar(500) DEFAULT NULL COMMENT '头像URL',
  `gender` tinyint DEFAULT NULL COMMENT '性别: 0-未知 1-男 2-女',
  `role` varchar(20) NOT NULL DEFAULT 'USER' COMMENT '角色: USER-普通用户 ADMIN-管理员',
  `status` tinyint NOT NULL DEFAULT 1 COMMENT '状态: 0-禁用 1-正常',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_phone` (`phone`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户信息表';
```

## 安全机制

1. **密码加密**：使用 BCrypt 加密存储
2. **Token 验证**：每次请求验证 JWT Token
3. **敏感词过滤**：用户输入进行敏感词过滤
4. **SQL 注入防护**：使用 MyBatis 参数化查询
